## Changes

### Change 1: Latency Thresholds for Stress/Spike Tests
Added P99 latency assertions to capture real performance regressions:
- `Transactions_StressTest`: P99 < 5000ms (5× baseline p99 of 1000ms)
- `Calendar_StressTest`: P99 < 10000ms (~3× baseline p99 of 3000ms)
- `Transactions_SpikeTest`: P99 < 8000ms (8× baseline p99 of 1000ms)

**Rationale:** Stress tests were only checking error rates. Without latency assertions, degradation under load goes undetected. These thresholds catch significant regressions while remaining permissive for in-memory/Testcontainers variance.

### Change 2: Relative Dates in Scenarios and Seeder
Replaced hardcoded date literals with `DateTime.UtcNow`-relative expressions:
- `TransactionsScenario`: queries `DateTime.UtcNow.AddMonths(-6)` → today
- `CalendarScenario` / `BudgetsScenario`: use current `DateTime.UtcNow.Year` / `Month`
- `TestDataSeeder`: seeds transactions 6 months ago → today to match scenario query range

**Rationale:** Hardcoded dates (`2025-09-01`, `2026-03-15`) drift from seeded data as time passes. By mid-2027, queries would find zero seeded data despite tests still passing (seeded data unchanged). Relative dates ensure performance tests always measure against a populated dataset.
