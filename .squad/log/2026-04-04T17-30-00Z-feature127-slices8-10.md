## Patterns Established This Session

1. **_Imports.razor global using:** Adding `@using` to `_Imports.razor` eliminates per-file boilerplate for all chart model types.
2. **Report page data availability constraint:** HeatmapChart, ScatterChart, CandlestickChart cannot be wired to existing reports without additional API calls — showcase only.
3. **SA1201 in code-behind pages:** Private state fields must precede `[Inject]` properties; SA1201 enforces this ordering.
4. **Dashboard loading state:** `_isLoading = true` before first API await; first call is the one used by loading-state test (`TaskCompletionSource` pin).
5. **Independent try-catch per data call:** Dashboard pages should degrade gracefully per chart rather than crashing entirely on any single API failure.
