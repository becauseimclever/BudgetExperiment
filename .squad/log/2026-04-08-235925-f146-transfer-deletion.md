## Key Architectural Decisions

1. **Atomic Delete vs Service Wrapper**
   - Repository returns `Task` (void response); service returns `Task<bool>` (existence check).

2. **Feature Flag Design**
   - Returns **403 Forbidden** (not 404) when disabled — signals feature exists but is gated.

3. **Orphan Handling**
   - Single leg deletion logged as warning; no exception thrown (graceful cleanup).

4. **Constructor Injection Changes**
   - Added `ILogger<TransactionRepository>` to repository constructor.
   - Required bulk update of all test instantiation sites.

5. **StyleCop Compliance**
   - SA1512: Section comments must not be followed by blank line.
   - SA1615: All public async methods must have `<returns>` in doc comments.
   - CS1734: `<paramref>` only for actual parameters.
   - SA1204: Static helpers before non-static in class sections.

## Lessons Learned

1. **NullLogger Injection in Tests**
   - When adding logger to repository, all test constructor calls must be updated.
   - Use Python regex for bulk updates (PowerShell regex inefficient on large files).

2. **Feature Flag Cache Invalidation**
   - Direct SQL updates without calling `SetFlagAsync` cause flaky tests.
   - Always use dedicated service method for cache-aware updates.

3. **StyleCop & XML Docs**
   - Document async methods with proper `<returns>` tags early (avoids batch rework).
   - Section comments conflict with immediate XML docs — remove section comments.

4. **Routing vs Model Binding**
   - `:guid` constraints return 404 (routing layer) not 400 (model binding).
   - Test expectations must match ASP.NET Core behavior, not assumptions.
